TIM SDK for iOS开发文档

 

版本: 1.0.0

 

简介

 

适配ios10.0以上

由于根据iOS系统的不断更新,最低版本支持10.0已经覆盖到绝大多数的设备；

不带UI的framework

我们根据是否带有UI功能，将不带UI的framework记为lib(TIMClientLib.framework); 提供基本的通信能力，包括消息的收发，接口的调用处理等.

带UI的framework

我们根据是否带有UI功能，将带UI的framework记为kit(TIMClientKit.framework); 提供直接接入app的显示能力，包括消息的收发页面的全部功能等.

#2.初始化

 

初始化TIMKit

需要引入头文件#import <TIMClientKit/TIMClientKit.h>

在全局类中使用(比如AppDelegate.m中)，需要有个TIMInitOption配置类，有个默认Option,再调用TIMKit的initWithOption接口即可。

xxxxxxxxxx
#import <TIMClientKit/TIMClientKit.h>
TIMInitOption *initOption = [[TIMInitOption alloc] defaultOption];
​
[[TIMKit sharedTIMKit] initWithOption:initOption];

 

#3.设置推送

设置推送接口

在application的didRegisterForRemoteNotificationsWithDeviceToken响应接口中调用；

xxxxxxxxxx
[[TIMKit sharedTIMKit] setDeviceTokenData:deviceToken];

 

#4.链接IM服务器

将通过 appServer 获取到的 userId和accessToken，连同在后台创建的appId一同初始化链接参数TIMConnectOption；

并调用链接接口连接IM服务器。

xxxxxxxxxx
TIMConnectOption * connectOption = [[TIMConnectOption alloc] initWithOption:@"您的appId" userId:@"您的userId" accessToken:@"您的accessToken"];
[[TIMKit sharedTIMKit] connect:connectOption success:^{
  // 链接成功
} error:^(TIMConnectErrorCode errCode, NSString * _Nonnull errorDes){
  // 链接失败 errCode为错误码 errorDes:为错误说明
} tokenIncorrect:^{
  // accessToken不正确
}

#5.设置监听

设置网络状态监听

xxxxxxxxxx
#pragma mark 连接状态监听
​
/**
 TIMKit连接状态的监听器
​
 @warning 如果您使用TIMKit，可以设置并实现此Delegate监听消息接收；
 */
-(void)setTIMKitConnectionChangeDelegate:(id<TIMConnectionStatusChangeDelegate>)delegate;

其中TIMConnectionStatusChangeDelegate定义在TIMClientLib.framework中，包含以下几种

xxxxxxxxxx
@protocol TIMConnectionStatusChangeDelegate <NSObject>
/*连接中*/
- (void)onConnecting;
/*连接成功*/
- (void)onConnected;
/*连接断开*/
- (void)onDisConnected;
/*断线重连中 round:断网后重连次数 interval:下次重连间隔(s)*/
- (void)onReConnecting:(int)round interval:(int)interval;
/*断线重连成功*/
- (void)onReConnected;
/*被踢下线 服务器产生*/
- (void)onKickOut;
@end

设置接收消息监听

xxxxxxxxxx
#pragma mark 消息接收监听
/**
 TIMKit消息接收的监听器
​
 @warning 如果您使用TIMKit，可以设置并实现此Delegate监听消息接收；
 */
 -(void)setTIMKitReceiveMessageDelegate:(id<TIMClientReceiveMessageDelegate>)delegate;

其中TIMClientReceiveMessageDelegate定义在TIMClientLib.framework中，包含以下几种

xxxxxxxxxx
@protocol TIMClientReceiveMessageDelegate <NSObject>
​
/**
 接收消息的回调方法
​
 @param message     当前接收到的消息
 @param nLeft       还剩余的未接收的消息数，left>=0
​
 @discussion 如果您设置了TIMlib消息监听之后，SDK在接收到消息时候会执行此方法。
 其中，left为还剩余的、还未接收的消息数量。比如刚上线一口气收到多条消息时，通过此方法，您可以获取到每条消息，left会依次递减直到0。
 您可以根据left数量来优化您的App体验和性能，比如收到大量消息时等待left为0再刷新UI。
 */
- (void)onReceived:(TIMMessage *)message left:(int)nLeft;

@optional
